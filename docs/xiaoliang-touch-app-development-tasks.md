# 小亮触摸屏应用生产开发任务

## 目标

面向 `xiaoliang_touch` 3.5 寸触摸屏，完成产品级应用中心。底部 `功能` 按钮进入 9 宫格，所有入口必须有完整页面、可点击交互、数据来源、空状态、错误状态和后续扩展接口。

默认硬件与资源约定：

- 默认板型：`CONFIG_BOARD_TYPE_XIAOLIANG_TOUCH`
- 音乐盒资源：`/sdcard/yinyuehe/manifest.json`
- 音频目录：`/sdcard/yinyuehe/audio`
- 当前宠物皮肤：`/sdcard/style/jinglingshu`
- MJPEG 文件名跨皮肤统一，只允许从 `/sdcard/style/<style_id>/<filename>.mjpeg` 读取

## 全局验收标准

- 触摸目标尺寸不小于 44px，高频按钮建议 48px 以上。
- 所有页面支持返回，不出现死路导航。
- 所有页面必须有真实数据、假数据或空状态，不允许空白页。
- 所有 SD 卡资源缺失时必须显示可理解的错误状态。
- 所有点击动作必须有反馈：页面跳转、状态更新或 notification。
- 页面不可遮挡状态栏、字幕栏和底部功能按钮。
- `idf.py build` 必须通过；至少对改动文件执行交叉编译器 `-fsyntax-only`。

## 生产交付范围

本轮目标不是只做入口，而是把触摸版 9 宫格中的每个应用做成可演示、可继续扩展的完整闭环。没有后端或真实数据的功能，先提供固定假数据、空状态、错误状态和待接入提示，但页面、跳转、按钮反馈必须完整。

交付优先级：

1. P0：应用中心框架、任务、萌宠乐园、音乐盒、资源路径规范。
2. P1：护眼岛、AI 听说、设备设置。
3. P2：健康提醒、表情相册、更多。
4. P3：代码拆分、通用组件沉淀、自动化测试补充。

每个应用必须包含：

- 首页概览：应用标题、关键数据、主操作。
- 二级页面：列表、详情或设置页，不能只停留在静态卡片。
- 操作反馈：点击后有页面变化、toast、弹窗或状态更新。
- 数据策略：真实服务优先，缺失时使用高质量假数据或空状态。
- 资源错误：SD 卡、manifest、MJPEG、音频缺失时给出明确提示。
- 扩展接口：新增模块、皮肤、音乐、场景时不需要重写主路由。
