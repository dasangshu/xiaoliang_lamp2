# 坐姿检测规则

本文档记录当前项目中的坐姿检测实现规则，主要对应：

- `main/application.cc`
- `main/application.h`
- `main/posture/posture_detector.cc`
- `main/posture/posture_detector.h`

## 启动条件

- 系统进入 `idle` 后等待 `10 秒` 才尝试启动坐姿检测。
- 只有满足以下条件才会启动：
  - 当前状态是 `idle`
  - WakeNet 正在运行
  - 音频队列空闲
- 启动坐姿检测前会停止 idle MJPEG 解码，但不显示摄像头画面。
- 摄像头采集使用 `CaptureSilent()`，即无感拍摄。

## 检测频率

- 摄像头每 `1.5 秒` 采集一帧。
- 模型输入尺寸为 `256x256`。
- 每次推理后额外延迟 `500ms`，让出 CPU/PSRAM 总线，优先保证 AFE/WakeNet。

## 坐姿类型

当前支持以下坐姿类型：

- `正常坐姿`
- `趴桌`
- `撑头`
- `弯腰驼背`
- `后仰`
- `身体倾斜`
- `未知状态`

## 姿态判断逻辑

- 使用 YOLO pose 关键点做人形检测。
- 人体检测置信度大于 `0.3` 才认为有效。
- 至少需要肩膀关键点，否则判为 `未知状态`。
- 头部中心使用鼻子、眼睛、耳朵的有效点平均值。
- 身体高度优先使用肩膀到臀部距离；臀部点缺失时，用人体框高度和肩宽兜底。

具体规则：

- `趴桌`
  - 有头部点时：头部相对肩膀明显下移，且左右偏移明显。
  - 无头部点时：使用肩膀在人体框中的位置兜底判断。
- `撑头`
  - 手腕距离头部较近，且不是明显后仰。
- `弯腰驼背`
  - 有头部点时：头部相对肩膀下移或前倾偏移。
  - 无头部点时：肩膀位置偏高，用人体框兜底判断。
- `身体倾斜`
  - 左右肩连线倾斜角度超过 `8°`。
- `后仰`
  - 有头部点时：头部相对肩膀明显上移。
- 以上条件都不满足时判为 `正常坐姿`。

## 提醒节奏

- 第一次识别到异常，只记录“异常开始”，不马上提醒。
- 异常持续 `5 秒`：
  - 触发轻提醒。
  - 屏幕显示文字建议。
  - 文字提醒 `4 秒` 后清除。
  - 健康值 `-1`。
- 异常持续 `12 秒`：
  - 触发语音提醒。
  - 两次语音提醒最短间隔为 `30 秒`。
  - 健康值 `-3`。
  - `弯腰驼背` 和 `趴桌` 使用 `pose2.ogg`。
  - 其他异常使用 `pose1.ogg`。
  - 提醒动作默认使用 `bye.mjpeg`。

## 恢复与健康值

- 从异常恢复到正常：
  - 健康值 `+2`。
  - 清空坐姿提示文字。
  - 如果之前已经提醒过，且距离上次恢复奖励超过 `30 秒`，显示“很好，坐姿恢复啦，继续保持”。
- 正常坐姿每 `60 秒` 健康值 `+1`。
- 连续良好坐姿 `3 分钟`：
  - 健康值 `+3`。
  - 显示良好坐姿奖励提示。
- 健康值范围固定在 `0 ~ 100`。

## 唤醒优先策略

唤醒模块是第一优先级。当前实现遵循以下降级策略：

- 进入聆听、说话、连接、启动、激活等状态都会停止坐姿检测。
- 坐姿提醒动画不会强行抢 WakeNet。
- 如果 WakeNet 正在运行，坐姿提醒会跳过 MJPEG 动画。
- 音频服务忙时不会播放坐姿提示音。
- 坐姿功能作为后台低优先级任务运行，不能压过唤醒和对话。
